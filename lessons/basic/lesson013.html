基礎編 第13回：Arduino本体のLEDを点滅させよう

Arduino基板の準備やソフトのインストールがまだの人へ：先に はじめる前の準備 を見てください。

1. テーマ

外付け部品を使わず、Arduino本体に付いているLED（オンボードLED）を、プログラムで点灯・消灯させて点滅させます。

2. 今回覚えること

💡 はじめに読むミニ知識：

ここに出てくる言葉は、このあとの説明やこれからのレッスンで何度も登場します。だから、今は「へぇ、そんな名前の言葉があるんだな」と眺めるだけで、完全に覚えられなくても全然大丈夫です！安心して進めてくださいね。

  • オンボードLED： Arduino本体にあらかじめ付いている小さなLEDのこと。外付け部品は使いません。
  • LED_BUILTIN： オンボードLEDにつながっているピンを表す名前。
  • pinMode（ピンモード）： ピンを「出力」や「入力」のどちらで使うかを決める関数。
  • OUTPUT（アウトプット）： ピンから信号を出す（出力する）設定。
  • digitalWrite（デジタルライト）： ピンを HIGH（高い）か LOW（低い）に切り替える関数。
  • HIGH / LOW： デジタル信号の2つの状態。多くのボードでは HIGH で点灯、LOW で消灯です。

3. 事前解説

ここまでの基礎編では、おもにシリアルモニタ（画面の文字）でプログラムの動きを見てきました。今回は、プログラムの結果が実物の光として見える回です。

朝の会（setup）で「このピンは出力用だよ」と申し送りし、授業（loop）で点灯→待つ→消灯→待つ、をくり返します。待つには、第3回で学んだ delay を使います。

注意： ボードの種類によって、オンボードLEDの場所や、HIGHで点くかLOWで点くかが違うことがあります。この講座では、Arduino Uno / Nano 系（多くの場合 HIGH で点灯）を想定します。光らないときはボードの説明も確認してください。

4. コード（プログラム）

初めての方・途中から始める方へ：インデントとカッコのルールを確認

下のアルファベットと記号を、パソコンの画面にそのまま打ち込んでみましょう。

void setup() {
  pinMode(LED_BUILTIN, OUTPUT);
}

void loop() {
  digitalWrite(LED_BUILTIN, HIGH);
  delay(1000);
  digitalWrite(LED_BUILTIN, LOW);
  delay(1000);
}

5. 動かし方（手順）

入力したプログラムを、実際に動かしてみましょう！

手順 1 / 準備
Arduinoをパソコンにつなぐ

USBケーブルを使って、Arduinoとパソコンをしっかりとつなぎます。ボードとポートが合っているかも確認します。

手順 2 / チェック
チェックボタン（検証）を押す

画面の左上にある「レ点（✓）のボタン」をクリックします。画面の下の方に「コンパイル完了」と出れば、間違いがないという証拠です！

手順 3 / 書き込み
矢印ボタン（書き込み）を押す

隣にある「右矢印（➔）のボタン」をクリックします。「書き込みが完了しました」と出るまで少し待ちます。

手順 4 / 確認
本体のLEDを見る

シリアルモニタは開かなくても大丈夫です。Arduino本体の小さなLEDが、およそ1秒ごとに点いたり消えたりするか見てください。

🔍 確認しよう：

オンボードLEDが、およそ1秒点灯・1秒消灯をくり返したら大成功です！

6. コード解説

  • void setup() { ... }
    朝の会。LED用ピンの準備を1回だけします。
  • pinMode(LED_BUILTIN, OUTPUT);
    LED_BUILTIN のピンを「出力（OUTPUT）」に設定します。まるカッコの中が、わたす材料です。
  • void loop() { ... }
    授業。点滅のくり返しはここで行います。
  • digitalWrite(LED_BUILTIN, HIGH);
    ピンを HIGH にします（多くのボードでは点灯）。
  • delay(1000);
    1000ミリ秒（1秒）待ちます。
  • digitalWrite(LED_BUILTIN, LOW);
    ピンを LOW にします（多くのボードでは消灯）。
  • ; と { }
    処理の区切りと、関数の範囲です。

7. 練習問題

2つの delay(1000); を delay(200); に変えて、もっと速い点滅になるか確かめてみましょう。終わったら、目にやさしい速さに戻すのも大切です。

8. うまく動かないとき（エラー）のアクション

もしうまく動かない場合は、あわてずに以下の 3つのポイント を手元と見比べて、直してみましょう。

1. ボードとポートは合っていますか？
書き込みできていないと、古いプログラムのまま動いていることがあります。
2. LED_BUILTIN のつづりは合っていますか？
大文字・小文字をまちがえると動きません。半角英数字で書いてください。
3. 見ているLEDはオンボードLEDですか？
電源ランプ（常時点灯）とまちがえないように、点滅している小さなLEDを探してください。ボードによっては場所が違います。
