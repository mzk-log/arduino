基礎編 第10回：そうでなければ（ if - else ）

1. テーマ

条件が正しいときの処理と、正しくないときの処理を分けて、どちらにも返事を返す方法を学びます。

2. 今回覚えること

💡 はじめに読むミニ知識：

ここに出てくる言葉は、このあとの説明やこれからのレッスンで何度も登場します。だから、今は「へぇ、そんな名前の言葉があるんだな」と眺めるだけで、完全に覚えられなくても全然大丈夫です！安心して進めてくださいね。

  • else： 「そうでなければ」という意味のキーワード。
  • 二者択一： 条件が正しいか正しくないかで、どちらか一方の道を進むこと。

3. 事前解説

第9回の if だけでは、「条件が合ったとき」しか反応しませんでした。今回は else をつけて、「合わなかったとき」の返事も書きます。

if (条件) { ... } else { ... } は、「もし条件が正しければ前者、そうでなければ後者」という意味です。

クイズで「正解」「不正解」の両方を返すイメージです。第12回のクイズ・ボットでも、この形を使います。

4. コード（プログラム）

初めての方・途中から始める方へ：インデントとカッコのルールを確認

下のアルファベットと記号を、パソコンの画面にそのまま打ち込んでみましょう。

void setup() {
  Serial.begin(9600);
  Serial.println("パスワードは Y です");
}

void loop() {
  if (Serial.available() > 0) {
    char c = Serial.read();

    if (c == 'Y') {
      Serial.println("正解！入れるよ");
    } else {
      if (c != '\n' && c != '\r') {
        Serial.println("違うよ。もう一度！");
      }
    }
  }
}

5. 動かし方（手順）

入力したプログラムを、実際に動かしてみましょう！

手順 1 / 準備
Arduinoをパソコンにつなぐ

USBケーブルを使って、Arduinoとパソコンをしっかりとつなぎます。

手順 2 / チェック
チェックボタン（検証）を押す

画面の左上にある「レ点（✓）のボタン」をクリックします。これは、プログラムに文字の間違いがないかをパソコンが自動で調べるボタンです。画面の下の方に「コンパイル完了」と出れば、間違いがないという証拠です！

手順 3 / 書き込み
矢印ボタン（書き込み）を押す

間違いがないことを確認したら、隣にある「右矢印（➔）のボタン」をクリックします。プログラムがArduinoへ送り込まれます。画面の下の方に「書き込みが完了しました」と出るまで少し待ちます。

手順 4 / 確認
シリアルモニタを開く

画面の右上にある「虫眼鏡（🔍）のボタン」をクリックして、シリアルモニタという新しい画面を開きます。

🔍 確認しよう：

「Y」を送ると「正解！入れるよ」、「Y」以外（例：N）を送ると「違うよ。もう一度！」と出たら大成功です。

6. コード解説

  • if (c == 'Y') { ... } else { ... }
    Y なら正解メッセージ、そうでなければ別メッセージを出します。
  • else
    直前の if の条件が正しくないときに実行されます。
  • c != '\n' && c != '\r'
    Enterキーなどで送られる改行文字のときは、エラーメッセージを出さないための工夫です。!= は「等しくない」、&& は「かつ」です。
  • Serial.available() / Serial.read()
    文字の到着確認と読み取りです。
  • void setup() / void loop() と ; { }
    朝の会・授業、区切り、範囲です。

7. 練習問題

正解の文字を 'Y' から 'G' に変えて、G のときだけ「正解」になるか確かめてみましょう。

8. うまく動かないとき（エラー）のアクション

もし画面にエラー（赤い文字など）が出て動かない場合は、あわてずに以下の 3つのポイント を手元と見比べて、直してみましょう。

1. else の位置は合っていますか？
if のなみカッコの直後に else が続く形です。
2. == と = をまちがえていませんか？
比べるときは == です。
3. なみカッコの数は足りていますか？
if と else と loop の { } を確認してください。
