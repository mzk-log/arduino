小・中学生のためのArduinoプログラミング講座

基礎編 第1回：パソコンの画面にメッセージを出してみよう！

1. テーマ

パソコンからArduino（アルドゥイーノ）へ文字データを送り、Arduinoがそれを受け取って、パソコンの画面にメッセージを表示する仕組みを学びます。

2. 今回覚えること

💡 はじめに読むミニ知識：

ここに出てくる言葉は、このあとの説明やこれからのレッスンで何度も登場します。だから、今は「へぇ、そんな名前の言葉があるんだな」と眺めるだけで、完全に覚えられなくても全然大丈夫です！安心して進めてくださいね。

  • 関数（かんすう）： 特定の仕事をまとめて実行するために、あらかじめ用意されている仕組みのこと。
  • void（ボイド）： 「空っぽ（何も返さない）」という意味の言葉。
  • 文字列（もじれつ）： " と " で囲まれた、ただの「文章・メッセージ」データのこと。
  • セミコロン（ ; ）： 処理の区切りにつける、日本語の「。」にあたる記号。
  • インデント（字下げ）： { の中身を右にずらして書くこと。仲間の行が一目で分かるようにする。
  • まるカッコ（ ( ) ）： 関数に材料をわたす場所。

3. 事前解説

プログラミングは、コンピューターに対して実行してほしい仕事を順番に書いていく作業です。この仕事のまとまりのことをプログラミングの世界では「関数（かんすう）」と呼びます。

Arduinoのプログラムには、最初から必ず setup（セットアップ） と loop（ループ） という名前の「２つの特別な関数」を書くルールになっています。

関数は仕事を終えると、たまに「計算結果の数字」などを返してくれることがありますが、setup と loop は仕事をやり遂げるだけで、何も結果を返しません。そのため、「この関数は何も返さないよ（空っぽだよ）」ということをコンピューターに教えるために、頭に void（ボイド） という言葉をつけます。

また、今回は画面に文字を表示させるために、すでに準備されている Serial.begin や Serial.println という関数も使います。画面に文字を表示するときは、データとして「これはプログラムの英語じゃなくて、ただの文章（文字列）だよ」と教えてあげる必要があります。そのため、画面に出したい日本語は "こんにちは" のように、必ず "（ダブルクォーテーション）という記号で挟むルールになっています。

コードを書くときは、インデント（字下げ）とカッコのちがいも大切です。くわしくは コードを書く基本ルール にまとめてあります。途中の回から始める人も、まずそちらを読んでください。

4. コード（プログラム）

初めての方・途中から始める方へ：インデントとカッコのルールを確認

下のアルファベットと記号を、パソコンの画面にそのまま打ち込んでみましょう。

void setup() {
  Serial.begin(9600);
  Serial.println("こんにちは");
}

void loop() {
}

5. 動かし方（手順）

入力したプログラムを、実際に動かしてみましょう！

手順 1 / 準備
Arduinoをパソコンにつなぐ

USBケーブルを使って、Arduinoとパソコンをしっかりとつなぎます。

手順 2 / チェック
チェックボタン（検証）を押す

画面の左上にある「レ点（✓）のボタン」をクリックします。これは、プログラムに文字の間違いがないかをパソコンが自動で調べるボタンです。画面の下の方に「コンパイル完了」と出れば、間違いがないという証拠です！

手順 3 / 書き込み
矢印ボタン（書き込み）を押す

間違いがないことを確認したら、隣にある「右矢印（➔）のボタン」をクリックします。プログラムがArduinoへ送り込まれます。画面の下の方に「書き込みが完了しました」と出るまで少し待ちます。

手順 4 / 確認
シリアルモニタを開く

画面の右上にある「虫眼鏡（🔍）のボタン」をクリックして、シリアルモニタという新しい画面を開きます。

🔍 確認しよう：

シリアルモニタの画面に「こんにちは」とパッと表示されたら大成功です！

6. コード解説

  • void setup() { ... }
    Arduinoの電源が入ったときに、上から順番に「最初の1回だけ」実行される特別な関数です。学校の「朝の会」のように、その日最初の準備をこの中でします。頭の void は「結果を返さない空っぽの関数」という意味です。
  • { と } （なみカッコ）
    この関数の仕事が「どこからどこまでなのか」という範囲を表す記号です。作文の段落のように、{ の中身は半角スペース2つ分だけ右にずらして書きます（インデント）。
  • ( と ) （まるカッコ）
    関数に「材料をわたす小さな箱」です。Serial.println("こんにちは") では箱の中に「こんにちは」を入れています。setup() のように空のときは、「わたす材料はないよ」という意味です。
  • Serial.begin(9600);
    パソコンとArduinoの間で、文字のやり取り（通信）ができるように準備を始める関数です。まるカッコの中の 9600 が、わたす材料（通信の速さ）です。
  • Serial.println("こんにちは");
    「こんにちは」という文字列データを、パソコンの画面（シリアルモニタ）に向けて送り出し、表示させるための関数です。
  • ; （セミコロン）
    「ここでひとつの処理が終わりです」という区切りの目印です。
  • void loop() { ... }
    setup（朝の会）が終わったあとに、ずーっと無限に何度もくり返し実行される特別な関数です。学校の「授業（じゅぎょう）」にあたる場所です。今回はまだ中に何も処理が書かれていないので、空っぽのままくり返します。

7. 練習問題

プログラムを書き換えて、パソコンの画面に出る文字を「こんにちは」から、夜のあいさつの「こんばんは」に変えてみましょう。

8. うまく動かないとき（エラー）のアクション

もし画面にエラー（赤い文字など）が出て動かない場合は、あわてずに以下の 3つのポイント を手元と見比べて、直してみましょう。

1. セミコロン（ ; ）を忘れていませんか？
Serial.println("こんばんは") の最後に ; があるか確認してください。
2. 記号が消えていませんか？
文字を挟む " と " 、関数を囲む { と } が、ちゃんと残っているか確認してください。
3. アルファベットは「半角（はんかく）」ですか？
英語の文字やカッコ、セミコロンが、日本語を入力するときの「全角（大きな文字）」になっていないか確認してください。（※プログラミングはすべて半角の英語や記号で書くルールです）
