基礎編 第2回：プログラムの２つの行事（朝の会 と 授業）

Arduino基板の準備やソフトのインストールがまだの人へ：先に はじめる前の準備 を見てください。

1. テーマ

Arduinoのプログラムには、最初に1回だけ動く関数と、そのあとずっとくり返す関数の2つがあります。それぞれの中に書いた文字データが、パソコンのシリアルモニタへどのように送られるかを学びます。

2. 今回覚えること

💡 はじめに読むミニ知識：

ここに出てくる言葉は、このあとの説明やこれからのレッスンで何度も登場します。だから、今は「へぇ、そんな名前の言葉があるんだな」と眺めるだけで、完全に覚えられなくても全然大丈夫です！安心して進めてくださいね。

  • setup（セットアップ）： 電源が入ったあと、最初に1回だけ実行される特別な関数。学校の「朝の会」にあたります。
  • loop（ループ）： setup が終わったあと、無限にくり返し実行される特別な関数。学校の「授業」にあたります。
  • 実行（じっこう）： プログラムに書かれた仕事を、コンピューターが実際に動かすこと。
  • くり返し： 同じ処理を、終わることなく何度も行うこと。

3. 事前解説

学校の1日を思い浮かべてみましょう。朝はまず「朝の会」があります。朝の会は、その日の最初に1回だけ行われます。朝の会が終わると、あとは「授業」が続きます。授業は、時間割のあいだずっとくり返されます。

Arduinoのプログラムも、これと同じ2つの行事でできています。

  • void setup() ＝ 朝の会（最初に1回だけ）
  • void loop() ＝ 授業（朝の会のあと、ずっとくり返す）

第1回では、setup（朝の会）の中だけにメッセージを書きました。今回は、loop（授業）の中にもメッセージを書いて、「1回だけ出る文字」と「何度も出る文字」のちがいを、シリアルモニタで確かめます。

なお、今回はまだ「待つ」命令（あとで学ぶ delay）を使わないので、loop の中の文字はとても速くくり返され、画面にたくさん並んで見えることがあります。それで正解です！

4. コード（プログラム）

初めての方・途中から始める方へ：インデントとカッコのルールを確認

下のアルファベットと記号を、パソコンの画面にそのまま打ち込んでみましょう。

void setup() {
  Serial.begin(9600);
  Serial.println("朝の会：準備完了！");
}

void loop() {
  Serial.println("授業：がんばっています");
}

5. 動かし方（手順）

入力したプログラムを、実際に動かしてみましょう！

手順 1 / 準備
Arduinoをパソコンにつなぐ

USBケーブルを使って、Arduinoとパソコンをしっかりとつなぎます。

手順 2 / チェック
チェックボタン（検証）を押す

画面の左上にある「レ点（✓）のボタン」をクリックします。これは、プログラムに文字の間違いがないかをパソコンが自動で調べるボタンです。画面の下の方に「コンパイル完了」と出れば、間違いがないという証拠です！

手順 3 / 書き込み
矢印ボタン（書き込み）を押す

間違いがないことを確認したら、隣にある「右矢印（➔）のボタン」をクリックします。プログラムがArduinoへ送り込まれます。画面の下の方に「書き込みが完了しました」と出るまで少し待ちます。

手順 4 / 確認
シリアルモニタを開く

画面の右上にある「虫眼鏡（🔍）のボタン」をクリックして、シリアルモニタという新しい画面を開きます。

🔍 確認しよう：

最初に「朝の会：準備完了！」が1回だけ出て、そのあと「授業：がんばっています」がずっとくり返し出てきたら大成功です！

6. コード解説

  • void setup() { ... }
    Arduinoの電源が入ったときに、最初の1回だけ実行される特別な関数です。学校の「朝の会」にあたります。頭の void は「結果を返さない空っぽの関数」という意味です。
  • { と } （なみカッコ）
    この関数の仕事が「どこからどこまでなのか」という範囲を表す記号です。setup にも loop にも、それぞれ1組ずつ必要です。
  • Serial.begin(9600);
    パソコンとArduinoの間で、文字のやり取り（通信）ができるように準備を始める関数です。朝の会（setup）の中で1回だけ行います。
  • Serial.println("朝の会：準備完了！");
    「朝の会：準備完了！」という文字列データを、シリアルモニタに表示する関数です。setup の中にあるので、1回だけ表示されます。
  • ; （セミコロン）
    「ここでひとつの処理が終わりです」という区切りの目印です。
  • void loop() { ... }
    setup（朝の会）が終わったあとに、ずーっと無限にくり返し実行される特別な関数です。学校の「授業」にあたります。
  • Serial.println("授業：がんばっています");
    「授業：がんばっています」という文字列データを、シリアルモニタに表示する関数です。loop の中にあるので、何度もくり返し表示されます。

7. 練習問題

loop（授業）の中の文字を、「授業：がんばっています」から、「授業：計算の練習中」に変えてみましょう。書き込み直したあと、シリアルモニタで「朝の会」は1回、「授業」はくり返し出るか確認してください。

8. うまく動かないとき（エラー）のアクション

もし画面にエラー（赤い文字など）が出て動かない場合は、あわてずに以下の 3つのポイント を手元と見比べて、直してみましょう。

1. setup と loop のなみカッコはそろっていますか？
setup 用の { } と、loop 用の { } が、それぞれ閉じているか確認してください。
2. セミコロン（ ; ）を忘れていませんか？
Serial.println(...) の最後に ; があるか確認してください。
3. アルファベットは「半角（はんかく）」ですか？
英語の文字やカッコ、セミコロンが、日本語を入力するときの「全角（大きな文字）」になっていないか確認してください。（※プログラミングはすべて半角の英語や記号で書くルールです）
