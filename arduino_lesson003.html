第3回：時間をあやつる命令を使おう（delay）

1. テーマ

メッセージを表示したあと、プログラムの流れをいったん止め、決めた時間だけ待ってから次の処理へ進む方法を学びます。

2. 今回覚えること

💡 はじめに読むミニ知識：

ここに出てくる言葉は、このあとの説明やこれからのレッスンで何度も登場します。だから、今は「へぇ、そんな名前の言葉があるんだな」と眺めるだけで、完全に覚えられなくても全然大丈夫です！安心して進めてくださいね。

  • delay（ディレイ）： プログラムを、決めた時間（ミリ秒）だけ一時停止する関数。
  • ミリ秒（ミリびょう）： 時間の単位で、1000ミリ秒＝1秒。
  • 待つ： 次の処理へ進む前に、時間を空けること。

3. 事前解説

第2回では、loop（授業）の中のメッセージがとても速くくり返されました。これは、コンピューターがとても速いスピードで仕事をしているからです。

そこで今回は、delay という関数を使います。delay(1000); と書くと、「ここで1000ミリ秒（＝1秒）待ってから次へ進んでね」という意味になります。

学校のたとえで言うと、授業の途中で「ちょっと一休み」を入れるイメージです。朝の会（setup）と授業（loop）のしくみはそのままです。

4. コード（プログラム）

初めての方・途中から始める方へ：インデントとカッコのルールを確認

下のアルファベットと記号を、パソコンの画面にそのまま打ち込んでみましょう。

void setup() {
  Serial.begin(9600);
  Serial.println("朝の会：準備完了！");
}

void loop() {
  Serial.println("授業：1秒ごとに表示");
  delay(1000);
}

5. 動かし方（手順）

入力したプログラムを、実際に動かしてみましょう！

手順 1 / 準備
Arduinoをパソコンにつなぐ

USBケーブルを使って、Arduinoとパソコンをしっかりとつなぎます。

手順 2 / チェック
チェックボタン（検証）を押す

画面の左上にある「レ点（✓）のボタン」をクリックします。これは、プログラムに文字の間違いがないかをパソコンが自動で調べるボタンです。画面の下の方に「コンパイル完了」と出れば、間違いがないという証拠です！

手順 3 / 書き込み
矢印ボタン（書き込み）を押す

間違いがないことを確認したら、隣にある「右矢印（➔）のボタン」をクリックします。プログラムがArduinoへ送り込まれます。画面の下の方に「書き込みが完了しました」と出るまで少し待ちます。

手順 4 / 確認
シリアルモニタを開く

画面の右上にある「虫眼鏡（🔍）のボタン」をクリックして、シリアルモニタという新しい画面を開きます。

🔍 確認しよう：

「朝の会：準備完了！」が1回出たあと、「授業：1秒ごとに表示」がおよそ1秒おきにくり返し出たら大成功です！

6. コード解説

  • void setup() { ... }
    最初に1回だけ実行される朝の会です。
  • Serial.begin(9600);
    パソコンとArduinoの通信準備です。
  • Serial.println("朝の会：準備完了！");
    朝の会のメッセージを1回表示します。
  • void loop() { ... }
    授業。終わると最初からくり返します。
  • Serial.println("授業：1秒ごとに表示");
    授業中のメッセージを表示します。
  • delay(1000);
    ここで1000ミリ秒（1秒）待ちます。待っているあいだは次の処理へ進みません。
  • ; （セミコロン）
    処理の区切りです。
  • { と }
    関数の範囲を表します。

7. 練習問題

delay(1000); を delay(2000); に変えて、およそ2秒ごとに表示されるか確かめてみましょう。

8. うまく動かないとき（エラー）のアクション

もし画面にエラー（赤い文字など）が出て動かない場合は、あわてずに以下の 3つのポイント を手元と見比べて、直してみましょう。

1. delay の数字は半角ですか？
delay(1000); の 1000 が全角になっていないか確認してください。
2. セミコロン（ ; ）を忘れていませんか？
delay(1000) の最後に ; があるか確認してください。
3. なみカッコはそろっていますか？
setup と loop の { } が閉じているか確認してください。
