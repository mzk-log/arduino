小・中学生のためのArduinoプログラミング講座

コードを書く基本ルール

プログラムを書くときに、毎回大切になるルールです。途中の回から始める人も、まずここを読んでからレッスンへ進みましょう。

1. インデント（字下げ）

作文で「段落の先頭を一字下げる」のと同じように、プログラムでも字を右にずらす書き方があります。これをインデントと呼びます。

なぜやるの？　{ と } のあいだにある「仲間の行」が、見ただけで分かるようにするためです。ずれていると、「どこからどこまでがひとまとまりか」が分かりにくくなり、} の付け忘れなどのミスにも気づきにくくなります。

この講座のルール：

  • { のあとに続く中身は、半角スペース2つ分だけ右にずらす
  • } は、ずらす前の位置（左）に戻す

良い例：

void setup() {
  Serial.begin(9600);
  Serial.println("こんにちは");
}

悪い例（中身がずらしていない）：

void setup() {
Serial.begin(9600);
Serial.println("こんにちは");
}
補足：

インデントがなくても、プログラムが動くことはあります。でも、読みやすさとミス防止のために、必ず守る習慣にしましょう。

2. カッコのちがい

プログラムには、見た目が似ていても役割がちがうカッコがあります。

{ }（なみカッコ）

「ここからここまでが、ひとまとまりの仕事です」という範囲を表します。関数の中身を囲むときに使います。

( )（まるカッコ）

関数に「材料をわたす小さな箱」だと思ってください。

  • Serial.println("こんにちは") … 箱の中に「こんにちは」を入れて、「これを表示してね」とわたしている
  • Serial.begin(9600) … 箱の中に数字 9600 をわたしている
  • setup() や loop() のように箱が空のとき … 「わたす材料はないよ」という意味

{ } が「仕事の囲い」なら、( ) は「材料をわたす場所」です。混ぜないようにしましょう。

3. ほかにも大切な記号

  • ;（セミコロン） … 日本語の「。」のように、「ここでひとつの処理が終わり」の目印
  • "（ダブルクォーテーション） … 文章（文字列）をはさむ記号
  • 半角 … プログラムの英語や記号は、すべて半角で書く
