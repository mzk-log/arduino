第5回：箱の中身を入れ替えよう（代入）

1. テーマ

すでに作った変数（箱）の中身を、あとから別の値に入れ替えて、画面に出す流れを学びます。

2. 今回覚えること

💡 はじめに読むミニ知識：

ここに出てくる言葉は、このあとの説明やこれからのレッスンで何度も登場します。だから、今は「へぇ、そんな名前の言葉があるんだな」と眺めるだけで、完全に覚えられなくても全然大丈夫です！安心して進めてくださいね。

  • 代入（だいにゅう）： すでに作った変数へ、新しい値を入れること。
  • =（代入演算子）： 「右の値を、左の変数へ入れる」記号。
  • 上書き： 箱の古い中身を消して、新しい中身にすること。

3. 事前解説

第4回では、変数を作ると同時に最初の値を入れました。今回は、そのあとに中身を入れ替える練習をします。

count = 5; のように書くと、「すでに作ってある count という箱に、あらためて 5 を入れる」という意味になります。これを代入と呼びます。

代入すると、箱の中の古い値は消えて、新しい値だけが残ります（上書き）。学校のノートに書いた答えを消して、新しい答えを書き直すイメージです。

4. コード（プログラム）

下のアルファベットと記号を、パソコンの画面にそのまま打ち込んでみましょう。

void setup() {
  Serial.begin(9600);

  int count = 1;
  Serial.println("最初の数");
  Serial.println(count);

  count = 5;
  Serial.println("入れ替えたあと");
  Serial.println(count);
}

void loop() {
}

5. 動かし方（手順）

入力したプログラムを、実際に動かしてみましょう！

手順 1 / 準備
Arduinoをパソコンにつなぐ

USBケーブルを使って、Arduinoとパソコンをしっかりとつなぎます。

手順 2 / チェック
チェックボタン（検証）を押す

画面の左上にある「レ点（✓）のボタン」をクリックします。これは、プログラムに文字の間違いがないかをパソコンが自動で調べるボタンです。画面の下の方に「コンパイル完了」と出れば、間違いがないという証拠です！

手順 3 / 書き込み
矢印ボタン（書き込み）を押す

間違いがないことを確認したら、隣にある「右矢印（➔）のボタン」をクリックします。プログラムがArduinoへ送り込まれます。画面の下の方に「書き込みが完了しました」と出るまで少し待ちます。

手順 4 / 確認
シリアルモニタを開く

画面の右上にある「虫眼鏡（🔍）のボタン」をクリックして、シリアルモニタという新しい画面を開きます。

🔍 確認しよう：

「最初の数」「1」「入れ替えたあと」「5」の順に表示されたら大成功です！

6. コード解説

  • int count = 1;
    変数 count を作り、最初に 1 を入れます。
  • Serial.println(count);
    いまの箱の中身を表示します（最初は 1）。
  • count = 5;
    すでに作った箱へ 5 を代入します。古い 1 は上書きされます。
  • =
    右の値を左の変数へ入れる記号です。
  • Serial.begin(9600);
    通信準備です。
  • void setup() / void loop()
    朝の会と授業です。今回の処理は朝の会で1回だけ行います。
  • ; と { }
    区切りと範囲を表す記号です。

7. 練習問題

2回目の代入を count = 5; から count = 100; に変えて、最後に「100」が出るか確かめてみましょう。

8. うまく動かないとき（エラー）のアクション

もし画面にエラー（赤い文字など）が出て動かない場合は、あわてずに以下の 3つのポイント を手元と見比べて、直してみましょう。

1. 変数を作る前に代入していませんか？
先に int count = 1; で箱を作ってから、あとで count = 5; します。
2. セミコロンを忘れていませんか？
count = 5 の最後に ; があるか確認してください。
3. 変数名のつづりは同じですか？
count と Count は別物として扱われます。
