小・中学生のためのArduinoプログラミング講座

第9回：もし〜なら（条件分岐 if）

1. テーマ

受け取った文字が特定の文字と一致するかどうかを調べ、条件が合ったときだけ特別なメッセージを出す方法を学びます。

2. 今回覚えること

💡 はじめに読むミニ知識：

ここに出てくる言葉は、このあとの説明やこれからのレッスンで何度も登場します。だから、今は「へぇ、そんな名前の言葉があるんだな」と眺めるだけで、完全に覚えられなくても全然大丈夫です！安心して進めてくださいね。

  • 条件分岐（じょうけんぶんき）： 条件によって、実行する処理を分けること。
  • if： 「もし〜なら」という意味のキーワード。
  • ==： 「左右が等しいか？」を調べる比較演算子（代入の = とは別物）。

3. 事前解説

第8回では、届いた文字をそのまま返しました。今回は「もしその文字が A なら、特別なメッセージを出す」という条件分岐を学びます。

if (条件) { 処理 } は、「もし条件が正しければ、なみカッコの中を実行する」という意味です。

大事なちがい：値を入れるときは =、比べるときは == を使います。ここをまちがえると、思った通りに動きません。

4. コード（プログラム）

初めての方・途中から始める方へ：インデントとカッコのルールを確認

下のアルファベットと記号を、パソコンの画面にそのまま打ち込んでみましょう。

void setup() {
  Serial.begin(9600);
  Serial.println("A を送ると反応します");
}

void loop() {
  if (Serial.available() > 0) {
    char c = Serial.read();

    if (c == 'A') {
      Serial.println("正解！ A を受け取りました");
    }
  }
}

5. 動かし方（手順）

入力したプログラムを、実際に動かしてみましょう！

手順 1 / 準備
Arduinoをパソコンにつなぐ

USBケーブルを使って、Arduinoとパソコンをしっかりとつなぎます。

手順 2 / チェック
チェックボタン（検証）を押す

画面の左上にある「レ点（✓）のボタン」をクリックします。これは、プログラムに文字の間違いがないかをパソコンが自動で調べるボタンです。画面の下の方に「コンパイル完了」と出れば、間違いがないという証拠です！

手順 3 / 書き込み
矢印ボタン（書き込み）を押す

間違いがないことを確認したら、隣にある「右矢印（➔）のボタン」をクリックします。プログラムがArduinoへ送り込まれます。画面の下の方に「書き込みが完了しました」と出るまで少し待ちます。

手順 4 / 確認
シリアルモニタを開く

画面の右上にある「虫眼鏡（🔍）のボタン」をクリックして、シリアルモニタという新しい画面を開きます。

🔍 確認しよう：

「A」を送ると「正解！ A を受け取りました」と出たら大成功です。別の文字ではこのメッセージは出ません。

6. コード解説

  • if (Serial.available() > 0)
    文字が届いているときだけ中に入ります。
  • char c = Serial.read();
    届いた1文字を読み取ります。
  • if (c == 'A')
    変数 c が文字 A と等しいかを調べます。
  • ==
    左右が等しいかを比べる記号です。代入の = とは違います。
  • 'A'
    文字1つはシングルクォーテーション ' で囲みます（文字列の " とは別）。
  • Serial.println("正解！ A を受け取りました");
    条件が正しいときだけ表示します。
  • { } と ;
    範囲と区切りです。

7. 練習問題

反応する文字を 'A' から 'B' に変えて、B を送ったときだけメッセージが出るか確かめてみましょう。

8. うまく動かないとき（エラー）のアクション

もし画面にエラー（赤い文字など）が出て動かない場合は、あわてずに以下の 3つのポイント を手元と見比べて、直してみましょう。

1. 比べるときは == ですか？
if (c = 'A') ではなく if (c == 'A') です。
2. 文字は 'A' ですか？
1文字は 'A'、文章は "A" です。混ぜないでください。
3. なみカッコはそろっていますか？
if が二重になっているので、{ } の数を数えて確認してください。
